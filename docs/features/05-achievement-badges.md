# 扩展功能五：SOS团成就徽章系统

## 功能背景

参考PlayStation Trophy、Steam成就、Xbox Gamerscore，以及二次元手游的「勋章墙」系统（如《原神》的「天地万象」、《明日方舟》的「蚀刻章」），SOS团也需要一套成就徽章系统来记录团员们的「丰功伟绩」。从「第一次签到」到「在无尽八月中存活」，每个徽章都是一段回忆。

> 「徽章？那种东西怎么样都好啦...不过既然你收集了这么多，我就勉为其难地表扬你一下吧。」——凉宫春日（好感度MAX限定语音）

## 徽章展示墙

### 页面布局
- 位于「我的档案」页面（或SOS团页面独立标签「徽章墙」）
- 整体布局参考PlayStation Trophy界面：
  - 顶部：用户头像 + 用户名 + 总徽章数 + 完成度百分比
  - 中部：稀有度筛选标签（全部/N/R/SR/SSR/UR/???）
  - 下部：徽章网格，已解锁彩色显示，未解锁灰色带问号遮罩

### 徽章卡片设计
- 尺寸：80x80px 正方形（桌面端）/ 60x60px（移动端）
- 形状：六边形（致敬《明日方舟》蚀刻章）或圆形
- 已解锁：彩色图标 + 稀有度边框特效 + 名称
- 未解锁：灰色剪影 + 黑色问号 + 「???」名称
- Hover/长按：展开详情弹窗，显示解锁条件和描述

### 总览面板

## 解锁动画

### 普通解锁
1. 屏幕右下角弹出小通知：「获得徽章：[徽章名]」
2. 通知带有徽章图标和稀有度颜色边框
3. 3秒后自动消失

### 稀有及以上解锁
1. 全屏暗化，中央弹出徽章大图
2. 徽章从模糊到清晰的聚焦动画
3. 稀有度文字逐字打出：「R A R E」
4. 对应稀有度的特效爆发（SR紫色闪电、SSR金色光柱、UR彩虹漩涡）
5. 显示徽章描述，底部有「分享」和「关闭」按钮

### 隐藏徽章解锁
1. 全屏显示静态雪花/噪点（模拟信号不良）
2. 中央慢慢浮现黑色徽章轮廓
3. 文字：「隐藏徽章已解锁...但你真的想知道它的名字吗？」
4. 点击后显示真实徽章信息

## 徽章分享卡

- 点击「分享」生成一张精美的成就分享图
- 分享图尺寸：1200x630px（适合社交媒体）
- 内容：
  - 顶部：SOS团标志
  - 中央：徽章大图 + 稀有度 + 名称
  - 底部：解锁日期 + 用户ID + 二维码（指向网站）
- 样式参考：PlayStation白金奖杯分享卡

## 视觉风格

- 徽章图标：简约线条风格，角色/事件抽象化符号
- 边框特效：
  - N：纯色细边框
  - R：蓝色微光边框
  - SR：紫色流光边框（CSS animation）
  - SSR：金色粒子环绕（CSS + 伪元素）
  - UR：彩虹全息效果（CSS conic-gradient + animation）
- 背景：深色玻璃态卡片（与SOS团明亮页面形成对比，突出徽章）

## 技术实现建议

### 数据结构
```typescript
interface Badge {
  id: string;
  name: string;
  description: string;
  rarity: 'N' | 'R' | 'SR' | 'SSR' | 'UR' | 'hidden';
  category: string;
  condition: string;      // 解锁条件描述（玩家可见）
  hiddenCondition?: string; // 实际判断逻辑
  icon: string;           // SVG路径或图标类名
  unlockedAt?: string;
}

interface UserBadgeState {
  badges: Record<string, boolean>;  // badgeId -> unlocked
  unlockHistory: Array<{ badgeId: string; unlockedAt: string }>;
}
```

### 解锁检测
- 在相关操作完成后调用 `checkBadgeUnlock()`
- 支持批量检测（页面加载时检查所有未解锁徽章的条件）
- 已解锁徽章状态存储在 `localStorage`，键名 `sos_badges`

### 隐藏徽章
- 隐藏徽章在列表中显示为黑色问号
- 解锁条件不在UI中显示，只有解锁后才知道是什么
- 可以设置「提示」功能：消耗贡献值获取一条模糊提示
