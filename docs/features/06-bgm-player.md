# 扩展功能六：SOS团活动室BGM播放器

## 功能背景

参考《Love Live!》的MV播放器、《BanG Dream!》的LIVE演出界面、《OSU!》的节拍点击，以及Bilibili音乐区的可视化播放器，SOS团活动室当然要有BGM。从《冒険でしょでしょ?》到《God knows...》，从《晴天愉快》到《優しい忘却》，每一首都是时代的记忆。

> 注意：由于版权问题，本播放器不播放真实音频，而是通过**歌词逐字显示 + 节拍可视化 + 专辑封面展示**来营造「正在播放」的氛围。用户可自行替换本地音频文件获得完整体验。

### 播放器模式

**模式A：歌词沉浸模式**
- 全屏显示歌词，逐字高亮（类似网易云音乐的歌词页面）
- 歌词使用角色代表色：春日黄、有希紫、实玖瑠粉
- 当前歌词字号放大，已唱歌词半透明，未唱歌词灰色
- 底部显示迷你播放控件

**模式B：可视化节拍模式**
- 中央显示专辑封面（角色立绘/动画截图占位图）
- 封面周围环绕频谱可视化条（Canvas/CSS模拟）
- 频谱颜色随曲目变化：God knows...是红色，晴天愉快是彩虹色
- 节拍点（BPM）触发封面轻微缩放脉冲

**模式C：SOS团LIVE模式**
- 模仿《BanG Dream!》的LIVE界面
- 底部出现节拍轨道，圆点从右向左移动
- 用户可以在特定节拍点点击屏幕「打call」
- 连击显示「COOL!」「GREAT!」「PERFECT!」评价
- 满分评价时显示角色头像应援动画

## 可视化效果详解

### 频谱条（Spectrum Bars）
使用CSS +少量JS实现，不需要真实音频分析：
```
20个竖条，每个高度随BPM随机波动
高度范围：10px ~ 80px
颜色渐变：根据曲目主题色
动画：高度变化使用 CSS transition，配合 BPM 节拍
```

### 专辑封面动效
- **播放中**：封面缓慢旋转（360度/20秒），带轻微呼吸缩放
- **暂停**：旋转停止，封面变暗（opacity: 0.7）
- **切歌**：旧封面向左滑出，新封面从右滑入（CSS transition）
- **节拍脉冲**：每拍封面轻微放大1.05倍后恢复

### 歌词逐字效果
- 当前歌词从灰色渐变为角色代表色
- 每个字延迟50ms依次变色，模拟「正在唱」的效果
- 歌词切换时，旧歌词向上淡出，新歌词从下方淡入
- 特殊歌词（如副歌）触发全屏粒子特效

## LIVE打call模式

### 玩法
1. 选择曲目后点击「进入LIVE」
2. 屏幕中央显示节拍轨道，圆点（notes）从右侧向左侧移动
3. 当圆点到达左侧判定线时，用户点击空格键或屏幕
4. 判定等级：
   - PERFECT（±50ms）：显示「PERFECT!」+ 角色应援头像
   - GREAT（±100ms）：显示「GREAT!」
   - GOOD（±150ms）：显示「GOOD」
   - MISS（>150ms）：显示「MISS...」，连击中断

### 连击系统
- 连续PERFECT时，屏幕顶部显示连击数
- 10连击：界面轻微震动
- 50连击：全屏闪光
- 100连击：触发「FULL COMBO」横幅
- ALL PERFECT（全曲无MISS）：触发「AP!」彩虹特效 + 获得隐藏徽章

### 难度分级
| 难度 | BPM倍速 | notes密度 | 适合曲目 |
|------|---------|-----------|----------|
| 简单 | 0.8x | 稀疏 | 雪、無音、窓辺にて。 |
| 普通 | 1.0x | 中等 | ハレ晴レユカイ |
| 困难 | 1.2x | 密集 | God knows... |
| 团长 | 1.5x | 地狱 | Super Driver |

## 视觉风格

- 播放器整体：深色玻璃态背景（rgba(20,20,40,0.8) + blur），与SOS团明亮页面形成「开灯/关灯」切换效果
- 按钮：圆形，带发光边框，Hover时放大
- 进度条：渐变色滑块，拖动时有弹性反馈
- 专辑封面：圆形裁剪，白色边框，带轻微阴影
- 整体色调：随当前曲目变化（深色底+曲目主题色点缀）

## 技术实现建议

### 数据结构
```typescript
interface Track {
  id: string;
  title: string;
  artist: string;
  duration: number;       // 秒
  bpm: number;
  themeColor: string;
  cover: string;          // 专辑封面路径
  lyrics: LyricLine[];
  visualizerStyle: string;
}

interface LyricLine {
  time: number;           // 该句开始时间（秒）
  text: string;
  character?: string;     // 角色代表色（可选）
}

interface PlayerState {
  currentTrackIndex: number;
  isPlaying: boolean;
  currentTime: number;
  volume: number;
  playMode: 'loop' | 'shuffle' | 'single';
  playlist: Track[];
}
```

### 虚拟播放逻辑
- 使用 `setInterval` 模拟播放进度（每100ms更新一次）
- 根据 `currentTime` 匹配当前歌词行
- 频谱条高度使用 `Math.random()` 结合BPM生成伪随机波形
- 所有时间基于预设的 `duration`，无需真实音频

### 音频接入（可选扩展）
- 预留 `<audio>` 标签接口
- 用户可将 `.mp3` 文件放入 `public/music/` 目录
- 在 `tracks.json` 中配置对应文件路径即可播放真实音频
- 默认状态显示「未检测到音频文件，当前为虚拟播放模式」
