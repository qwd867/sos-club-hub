# SOS团招新页面设计方案

## 项目背景
将登录后的 `Demo` 页面改造为 **SOS团官方招新主页**，作为用户登录后的首个着陆页。风格延续前端的二次元玻璃态设计体系，但切换为更明亮、活力、社团宣传感强烈的视觉氛围。

---

## 设计目标
1. 营造SOS团特有的热闹、神秘、充满可能性的社团氛围
2. 通过角色展示增强IP代入感，让用户产生"想要加入"的情感共鸣
3. 保持与登录/注册页面一致的UI语言（玻璃态、渐变、微动画），建立品牌连贯性
4. 提供装饰性互动元素（申请表单、时间轴），增加页面趣味性和停留时长

---

## 整体视觉风格

### 参考对象
| 参考类型 | 具体网站/产品 | 借鉴点 |
|----------|---------------|--------|
| 活动页设计 | Bilibili 专题活动页 | 大色块分割、卡片式角色展示 |
| 角色展示 | Galgame 官网角色页 | 立绘展示、台词浮动、详情展开 |
| 排版气质 | 京都动画官网 | 简洁留白、柔和渐变、高级灰调 |

### 色彩方案
- **主色**：SOS团黄 `#FFD700`（春日发带色）
- **辅色**：资讯蓝 `#00BFFF`（长门有希/资讯统合思念体）
- **点缀**：实玖瑠粉 `#FF69B4`
- **背景基底**：浅蓝紫渐变 `linear-gradient(135deg, #e0e7ff 0%, #f3e8ff 50%, #fff0f5 100%)`
- **玻璃态卡片**：`rgba(255, 255, 255, 0.6)` + `backdrop-filter: blur(16px)`

### 背景效果
- 明亮的动态渐变背景（比登录页浅，营造白天/社团活动室氛围）
- 稀疏的浮动粒子（金色+蓝色光点，缓慢飘动）
- 可选： subtle 的SOS团标志水印纹理

## 页面结构（共5大区块）

### 区块1：Hero英雄区
**位置**：页面首屏

**内容**：
- 左侧/中央：SOS团标志（大字号文字动画或SVG徽章）
- 主标语：`让世界变得更热闹的凉宫春日的团`
- 副标语：`—— 北高一年级神秘组织 · 现在招新中！——`
- CTA按钮：`我要入团`（带春日黄渐变+悬停发光）
- 右侧/背景：半透明角色剪影组合（五人合照风格，低透明度作为装饰）

**动效**：
- 团名文字逐个字符淡入
- CTA按钮轻微上下浮动
- 背景渐变缓慢流动

**卡片交互**：
- **默认状态**：角色半身像 + 名字 + 身份标签 + 代表色发光边框
- **Hover效果**：卡片轻微上浮（translateY -8px）、边框发光增强、背景出现角色代表色渐变
- **点击效果**：弹出全屏玻璃态模态框（Modal）
  - 模态框左侧：角色大图
  - 模态框右侧：详细分类介绍（角色定位/团内职责/经典台词/特殊能力/萌点）
  - 关闭方式：点击背景X按钮或ESC键

   - ○ 我相信外星人存在！（长门派）

   - ○ 我相信超能力是真的！（古泉派）

   - 想要改变世界
   - 想要交朋友
   - 被迫的（春日强迫）
   - 只是来看实玖瑠的
   - 其他

**提交后效果**：
- 按钮变为加载状态
- 弹出春日团长的批准动画（春日头像 + 「合格！你被录取了！」）

## 技术实现建议

### 组件拆分
```
views/
  Demo.vue              # 主页面，组装所有区块
components/
  HeroSection.vue       # 英雄区
  MemberCard.vue        # 单个角色卡片
  MemberDetailModal.vue # 角色详情弹窗
  Timeline.vue          # 时间轴
  TimelineItem.vue      # 单个时间轴节点
  JoinForm.vue          # 入团申请表单
  FooterSection.vue     # 页脚
layouts/
  AuthLayout.vue        # 已有（登录/注册/忘记密码）
```

### 动画库建议
- **Vue内置**：`<Transition>` 和 `<TransitionGroup>` 处理列表动画和淡入淡出
- **CSS动画**：卡片浮动、背景渐变流动、粒子飘动
- **Intersection Observer API**（原生）：时间轴滚动触发，无需引入额外库
- **可选增强**：如需复杂时间轴动画，可引入 `gsap`，但建议优先用CSS实现

### 图片资源
- 角色图：使用萌娘百科公开图片作为占位（已有春日头像链接，其他角色同来源获取）
- 用户后续可替换为自己收藏的高清立绘
- 所有图片使用 `object-fit: cover` 确保不同比例图片显示正常
